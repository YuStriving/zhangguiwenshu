# 🚀 从零搭建智能数据查询 Agent 系统指南

## 📋 指南概述

本文档提供从零开始搭建智能数据查询 Agent 系统的完整步骤，适合零基础开发者学习。我们将从环境准备开始，逐步完成系统搭建和部署。

## 🛠️ 环境准备

### 1. 系统要求
- **操作系统**: Windows 10/11, macOS 10.15+, Linux Ubuntu 18.04+
- **内存**: 至少 8GB RAM
- **存储**: 至少 20GB 可用空间
- **网络**: 稳定的互联网连接

### 2. 软件安装

#### 2.1 安装 Python 3.11+
```bash
# Windows: 从官网下载安装包
# macOS: brew install python@3.11
# Linux: sudo apt install python3.11 python3.11-pip

# 验证安装
python --version  # 应该显示 Python 3.11.x
pip --version     # 应该显示 pip 版本
```

#### 2.2 安装 Docker 和 Docker Compose
```bash
# Windows: 安装 Docker Desktop
# macOS: 安装 Docker Desktop
# Linux: 
sudo apt update
sudo apt install docker.io docker-compose

# 验证安装
docker --version
docker-compose --version
```

#### 2.3 安装 Git
```bash
# Windows: 安装 Git for Windows
# macOS: brew install git
# Linux: sudo apt install git

# 验证安装
git --version
```

## 🔧 常见问题解决

### 问题 1: Docker 服务启动失败

**症状**: `docker-compose up -d` 命令失败

**解决方案**:
```bash
# 检查 Docker 服务状态
sudo systemctl status docker

# 重启 Docker 服务
sudo systemctl restart docker

# 清理 Docker 缓存
docker system prune -a

# 重新启动服务
docker-compose down
docker-compose up -d
```

### 问题 2: Python 依赖安装失败

**症状**: `pip install` 命令报错

**解决方案**:
```bash
# 更新 pip
pip install --upgrade pip

# 使用国内镜像源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 如果仍有问题，逐个安装依赖
pip install fastapi uvicorn loguru
```

### 问题 3: 数据库连接失败

**症状**: 应用无法连接到 MySQL

**解决方案**:
```bash
# 检查 MySQL 服务状态
docker ps | grep mysql

# 查看 MySQL 日志
docker logs docker-mysql-1

# 重新启动 MySQL 服务
docker restart docker-mysql-1
```

### 问题 4: 向量数据库连接失败

**症状**: Qdrant 服务无法访问

**解决方案**:
```bash
# 检查 Qdrant 服务状态
docker ps | grep qdrant

# 查看 Qdrant 日志
docker logs docker-qdrant-1

# 重新创建 Qdrant 集合
curl -X DELETE http://localhost:6333/collections/columns
curl -X DELETE http://localhost:6333/collections/metrics
```

## 📊 性能优化建议

### 1. 硬件优化
- **增加内存**: 建议 16GB+ RAM 以获得更好性能
- **使用 SSD**: 固态硬盘可显著提升 I/O 性能
- **多核 CPU**: 支持更好的并发处理

### 2. 软件优化
- **调整 Docker 资源限制**: 为容器分配更多资源
- **优化数据库配置**: 调整 MySQL 缓存设置
- **启用缓存**: 使用 Redis 缓存频繁查询结果

### 3. 网络优化
- **使用内网**: 数据库服务部署在同一网络
- **CDN 加速**: 前端资源使用 CDN 加速
- **负载均衡**: 多实例部署实现负载均衡
